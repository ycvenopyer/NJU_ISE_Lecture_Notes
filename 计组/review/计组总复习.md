# 一.计算机系统概述

1. 计算机硬件的基本组成？

   **冯·诺依曼结构**：计算机由运算器、控制器、存储器、输入设备、输出设备构成。

   **现代计算机**：把运算器、控制器和各类寄存器等互连在一个中央处理器中(CPU)。

   **中央处理器**：核心部件，用于指令执行；包含数据通路和控制器。

   **存储器**：分为内存和外存。

   **外部设备和设备控制器**：外设通过设备控制器连接到主机上。

   **总线**：传输信息的介质，用于在部件之间传输信息。

   应用软件：专门为某种应用编写的程序

   系统软件：为有效、安全地使用和管理计算机以及为开发和运行应用软件而提供的各类软件，介于计算机硬件与应用软件之间；操作系统、语言处理系统等

2. 原码、补码、译码表示？

   原码：直观方便

   **补码**：模运算，实现+和-的统一

   一个负数的补码等于模减去该负数的绝对值

   对于某一确定的模，数x减去小于模的数y，总可以用数x加上-y的补码来代替

   正数：符号为0，数值部分是它本身

   负数：模与该负数绝对值之差

   $[X_T]$补$=M+X_T(mod \quad M)$

   一个负数的补码等于对应正数补码的各位取反、末位加一

   变形补码

   ![Quicker_20240101_161257](C:\Users\yangchen\Desktop\Ehereal_YC\CodeBase\Markdown\picture\Quicker_20240101_161257.png)

   反码：负数的各位取反但不加1

   **移码**：将每个数值加上一个偏置常数

5. 浮点数加减运算、舍入方式？

   （1）加减：

   1. 对阶：小阶向大阶看齐，阶小的那个数的尾数右移，右移的位数等于两个阶的差的绝对值

      当$\Delta E$溢出时不能用来判断阶差

   2. 尾数加减：隐藏位还原后，按原码进行加减运算，附加位一起运算

   3. 尾数规格化：当尾数高位为0，则需左规：尾数左移一次，阶码减1，直到MSB为1；当尾数最高位有进位，需右规：尾数右移一次，阶码加1，直到MSB为1，每次阶码加1后要判断阶码是否上溢

# 四.指令系统

1. 指令格式设计？

   **一条指令包含的信息**：操作码、源操作数或其地址、结果的地址、下条指令的地址（通常隐含在PC中）

   **操作码**：定义操作类型

   **操作数**：表示操作的源和目的

   **指令地址码的个数**：

   - 零地址指令：无需操作数，所需操作数为默认的
   - 一地址指令：其地址既是操作数的地址，也是结果的地址
     - 单目：取反/取负等
     - 双目：另一操作数默认，如：累加器
   - 二地址指令（最常用）：分别存放双目运算中两个操作数，其中一个作为结果地址
   - 三地址指令（RISC风格）：分别作为双目运算中两个源操作数的地址和一个结果的地址
   - 多地址指令：用于成批数据处理的指令

   **指令格式的选择原则；**

   - 应尽量短
   - 要有足够的操作码位数
   - 指令编码必须有唯一的解释，否则是不合法的序列
   - 指令字长应是字节的整数倍
   - 合理地选择地址字段的个数
   - 指令尽量规整

   **指令系统设计**原则：完备性或完整性、兼容性、均匀性、可扩充性

3. 异常和中断的区别?

   （1）异常（例外）：是指处理器在执行某条指令时发生在CPU内部的事件。如整除0、溢出、断点设置、单步跟踪、访问超时、非法操作码、栈溢出、缺页、保护错等。内部异常可分为故障、自陷、终止。

   （2）中断：程序执行过程中，若外设完成任务或发生某些特殊事件，会向CPU发中断请求，要求CPU对这些情况进行处理。

   （3）比较：中断是一种典型的由I/O设备触发的、与当前正在执行的指令无关的异步事件；而异常是处理器执行一条指令时，由处理器在其内部检测到的、与正在执行的指令相关的同步事件。

   （4）阶段：

   检测和响应：由硬件完成

   具体的处理过程由软件（OS）执行程序完成

6. 过程调用的指令、执行步骤、栈和栈帧的变化？

   （1）过程调用：

   **MIPS中用于过程调用的指令:**

   **jal:J型指令**，两个功能：保存下一条指令到31号寄存器中，跳转到指定地址处执行

   **jr:R型指令，**功能：跳转到寄存器指定的地址处执行

   

   **过程调用时MIPS寄存器的使用约定：**

   假定过程P调用过程Q，则过程调用的执行步骤如下；

   1. 将**参数**放到Q能访问到的地方
   2. 将P中的**返回地址**存到特定的地方，将控制转移到过程Q
   3. 为Q的局部变量分配空间（局部变量临时保存在栈中）
   4. 执行过程Q
   5. 将Q执行的**返回结果**放到P能访问到的地方
   6. 取出返回地址，将控制转移到P，即返回到P中执行

   

   **栈的使用**！

   一般用栈来传递参数、保存返回地址，并用来临时存放过程中的局部变量等，因为这样做**便于嵌套或递归调用**

   （2）栈的概念：

   先进后出的队列

   需要一个栈指针指向栈顶元素

   每个元素长度一致

   用入栈和出栈操作访问栈中元素

   （3）MIPS中栈的实现：

   用栈指针寄存器（$sp）来指示栈顶元素

   每个元素的长度为32位，即一个字（4个字节）

   入栈和出栈用sw和lw实现，需要用add/sub指令调整栈指针的值，不能自动进行栈指针的调整

   栈生长的方向：从高地址向低地址增长，而取数/存数的方向相反

   每入栈1字，$\$sp-4->\$sp$

   ![Quicker_20240102_152342](C:\Users\yangchen\Desktop\Ehereal_YC\CodeBase\Markdown\picture\Quicker_20240102_152342.png)

   （4）栈帧：

   各过程有自己的栈区，称为栈帧（Stack frame）,即过程的帧，栈由若干栈帧组成。

   有专门的帧指针寄存器指定起始位置。

   当前栈帧范围在帧指针和栈指针之间。

   程序执行时，栈指针可移动，帧指针不变。所以过程中对栈信息的访问可以通过帧指针进行。

   复杂局部变量一定分配在栈帧中。

6. 多周期数据通路设计？能够理解多周期数据通路设计，明确控制信号取值、指令执行状态转换图。

   （1）单周期处理器的缺陷：

   时钟周期远远大于其他指令实际所需的执行时间，效率极低

   （2）多周期处理器设计思想：

   把指令的执行分成多个阶段，每个阶段在一个时钟周期内完成

   - 时钟周期以最复杂阶段所花的时间为准

   - 尽量分成大致相等的若干阶段

   - 规定每个阶段最多只能完成1次访存或寄存器堆读/写或ALU

   每步都设置存储元件，每步执行结果都在下个时钟开始保存到相应单元

   **优点**；

   - 时钟周期短

   - 不同指令所用周期数可以不同

     load5个CLK，Jump3个CLK

   - 允许功能部件在一条指令执行过程中被重复使用

     单周期有Adder和ALU，多周期只用一个ALU，在不同周期可重复使用；单周期分开指令和数据存储器，多周期时合用，不同周期重复使用

   （3）多周期数据通路的设计

   **竞争问题：**事实上，存储器的实际写操作不是由时钟边沿触发的，而是一个组合逻辑电路。写操作为：当WE信号有效，并且写入数据和地址已经稳定，则经过一个写操作时间后，数据被写入。这里，重要的一点是地址和数据必须在WE信号有效前先稳定在各自的输入端。因此，实际的存储器在单周期数据通路中不能可靠工作，这是因为不能保证地址和数据能在WE信号有效前稳定，即地址、数据和信WE信号之间存在竞争问题。

   **解决方法**：首先确定地址和数据在第n周期结束时已经稳定，然后，使WE信号在一个周期后有效，并使地址和数据在WE信号无效前不改变其值。

# 七.存储器层次结构

1. 存储器的分类、主存储器的组成和基本操作、存储器的层次化结构？

   （1）分类：

   按工作性质/存取方式分类

   - **随机存取存储器 Random Access Memory (RAM)**：按地址访问，每个单元读写时间一样，且与各单元所在位置无关。如：内存。（注：原意主要强调地址译码时间相同。现在的DRAM芯片采用行缓冲，因而可能因为位置不同而使访问时间有所差别。）

   - **顺序存取存储器 Sequential Access Memory (SAM)：**数据按顺序从存储载体的始端读出或写入，因而存取时间的长短与信息所在位置有关。例如：磁带。

   - **直接存取存储器 Direct Access Memory (DAM)：**直接定位到读写数据块，在读写数据块时按顺序进行。如磁盘。

   - **相联存储器 Associate Memory (AM)**，Content Addressed Memory (CAM)：按内容检索到存储位置进行读写。例如：快表

   按存储介质分类

   - **半导体存储器：**双极型，静态MOS型，动态MOS型

   - **磁表面存储器**：磁盘（Disk）、磁带 （Tape）

   - **光存储器**：CD，CD-ROM，DVD

   按信息的可更改性分类

   - **读写存储器（Read / Write Memory)**：可读可写

   - **只读存储器（Read Only Memory)：**只能读不能写

   按断电后信息的可保存性分类

   - **非易失（不挥发）性存储器(Nonvolatile Memory)** ：**信息可一直保留， 不需电源维持。（如 ：ROM、磁表面存储器、光存储器等）**

   - **易失（挥发）性存储器(Volatile Memory)：电源关闭时信息自动丢失。（如：RAM、Cache等）**

   按功能/容量/速度/所在位置分类

   - **寄存器(Register)：**封装在CPU内，用于存放当前正在执行的指令和使用的数据；用**触发器实现，速度快，容量小**（几~几十个）

   - **高速缓存(Cache)**：位于CPU内部或附近，用来存放当前要执行的局部程序段和数据；用**SRAM**实现，速度可与CPU匹配，容量小（几MB）

   - **主存储器MM**（Main (Primary) Memory）：位于CPU之外，用来存放已被启动的程序及所用的数据；用**DRAM**实现，速度较快，容量较大（几GB）

   - **外存储器AM (**辅助存储器Auxiliary / Secondary Storage)：位于主机之外，用来存放暂不运行的程序、数据或存档文件；用磁表面或光存储器实现，容量大而速度慢

9. Cache的基本工作原理？

   Cache是一种小容量的高速缓冲存储器，由SRAM组成。Cache直接制作在CPU内，速度几乎一样快。程序运行时，CPU使用的一部分数据/指令会预先成批拷贝在Cache中，Cache的内容是主存储器中部分内容的映像。当CPU需要从内存中读/写数据或指令时，先检查Cache，有着直接访问Cache，而不用访问MM

10. 直接映射、全相连映射、组相连映射？（命中率、命中时间、缺失损失、平均访问时间）

    三种映射方式具体见课本。

    （1）命中率、缺失率、缺失损失

    - 命中 Hit: 要访问的信息在Cache中

      Hit Rate(命中率)：在Cache中的概率

      Hit Time (命中时间) ：在Cache中的访问时间，包括：

      Time to determine hit/miss + Cache access time 

      (即： 判断时间 + Cache访问)

    - 缺失 Miss: 要找的信息不在Cache中

      Miss Rate (缺失率) = 1 - (Hit Rate)

      Miss Penalty (缺失损失)：访问一个主存块所花时间

    （2）平均访问时间：

    命中率$\alpha$,缺失率$1-\alpha$

    平均访问时间:$t_{avg}=\alpha t_c+(1-\alpha)(t_c+t_m)=t_c+(1-\alpha)t_m$

    （3）三种映射方式和比较：

    **关联度**：一个主存块映射到Cache中时，可能存放的位置的个数

    - 直接映射关联度：1，最低

    - 全相联映射关联度：Cache行数，最高

    - N-路组相联映射关联度：N

    - 缺失率：直接映射最高，全相联映射最低

    - 命中时间：直接映射最小，全相联映射最大

    标记位大小与关联度：关联度越高，总的标记位数越多，额外空间开销越大！

12. 全写法、回写法的区别？

    Cache的一致性问题：

    为什么要保持在Cache和主存中的数据一致？

    因为Cache中的内容是主存块的副本，当对Cache中的内容进行更新时，就存在Cache和主存如何保持一致性的问题

    - 写操作分为写命中：要写的单元已经在Cache中了和写不命中；要写的单元不在Cache中

    - 处理方法：

      对于写命中：

      - 全写法(write through,通过式写、写直达、直写)

        同时写Cache和主存单元

        使用写缓冲

      - 回写法(write back,一次性写、写回、回写)

        只写Cache不写主存，缺失时一次写回，每行有一个修改位（dirty bit），大大降低主存带宽需求，控制可能很复杂

      对于写不命中：

      - 写分配 Write Allocate

        将主存块装入Cache，然后更新相应单元

        试图利用空间局部性，但每次都要从主存读一个块

      - 非写分配 Not Write Allocate 

        直接写主存单元，不把主存块装入到Cache

    > 直写Cache可用非写分配或写分配
    >
    > 回写Cache通常用写分配

# 八.系统互连及输入输出组织

1. 外设的分类？：

   分类：

   - 按信息的传输方向来分：

     **输入设备**：键盘、鼠标、扫描仪等

     **输出设备**：打印机、显示器等

     **输入/输入出设备**：磁盘驱动器、光盘驱动器、CRT终端、网卡之类的通信设备等。

   - 按功能来分：

     **人机交互设备**：用于用户和计算机之间交互通信的设备。(如：键盘、鼠标、扫描仪、打印机、显示器等)

     **存储设备**：用于存储大容量数据，作为计算机的外存储器使用。(如：磁盘驱动器、光盘驱动器等)

     **机-机通信设备**：用于计算机及和计算机之间的通信。(如：网卡、调制解调器、数/模和模/数转化设备等)

   特点：

   - **异步性：**外设与CPU之间是完全异步的工作方式，两者之间无统一的时钟。

   - **实时性：**CPU必须及时按不同的传输速率和不同的传输方式接收来自多个外设的信息或向多个外设发送信息，否则高速设备可能丢失信息。

   - **多样性：**外设的多样性造成了主机与外设之间连接的复杂性。为简化控制，计算机系统往往提供标准接口，以便各类外设通过自己的设备控制器与标准接口相连。

2. 总线、系统总线、数据线、地址线、控制线？

   **总线**是计算机内**数据传输的公共路径**，用于实现两个或两个以上部件之间的信息交换。

   **系统总线**指连接处理器芯片、存储器芯片和各种I/O模块等主要部件的总线。**系统总线**通常由**一组控制线、一组数据线和一组地址线**构成。也有些总线没有单独的地址线，地址信息通过数据线来传送，这种情况称为**数据/地址复用。**

   - **数据线**（Data Bus）：承载在源和目部件之间传输的信息。数据线的宽度反映一次能传送的数据的位数。

   - **地址线**（Address Bus） ：给出源数据或目的数据所在的主存单元或I/O端口的地址。地址线的宽度反映最大的寻址空间。

   - **控制线**（Control Bus） ：控制对数据线和地址线的访问和使用。用来传输定时信号和命令信息。

5. I/O端口的独立编址方式、统一编址方式？

   （1）独立编址方式：

   不和主存单元一起编号，单独编号。需要专门的I/O指令

   （2）统一编址方式：

   与主存空间统一编址，将主存空间分出一部分地址给I/O端口进行编号

   将I/O端口映射到某主存空间中，也称为存储器映射方式

6. 程序直接控制I/O方式、中断控制I/O方式、DMA方式的工作原理、区别？

   （1）程序直接控制方式（最简单的I/O方式）

   无条件传送：对简单外设定时（同步）进行数据传送

   条件传送：Polling(轮询，查询)：OS主动查询，也称为程序查询方式

   - I/O设备将自己的状态放到一个状态寄存器中
   - OS阶段性地查询状态寄存器中的特定状态，以决定下一步的动作

   （2）I/O Interrupt（中断I/O方式）：几乎所有系统都支持的中断I/O方式

   - 若一个I/O设备需要CPU干预，它就通过中断请求通知CPU
   - CPU中止当前程序的执行，调出OS（中断处理程序）来执行
   - 处理结束后，再返回到被中止的程序继续执行
   - **OS是被动调出的，也称为中断驱动I/O方式**

   （3）直接存储器存取（DMA方式）：磁盘等高速外设特有的I/O方式

   - 磁盘等高速外设成批地直接和主存进行数据交换
   - 需要专门的DMA控制器控制总线，完成数据传送
   - 当外设准备好数据后，向DMA控制器发出DMA请求信号，DMA控制器再向CPU发出总线请求，CPU让出总线后，由DMA控制器控制总线进行传输，无需CPU干涉

8. 3种DMA方式：CPU停止法、周期挪用法、交替分时访问法？

   DMA：直接存储器存取

   基本思想：在高速外设和主存间直接传送数据，由专门硬件（DMA接口）控制总线进行传输

   适用：高速设备，大量成批数据

   采用“请求-响应”方式

   与中断控制方式结合使用

   （1）CPU停止法（成组传送）：

   DMA传输时，CPU脱离总线，停止访问MM，直到DMA传送数据结束

   （2）周期挪用法（单字传送）：DMA传输时，CPU让出一个总线事务周期，由DMA控制总线来访问MM，传送完一个数据后立即释放总线

   （3）交替分时访问法：

   每个存储周期分成两个时间片，一个给CPU，一个给DMA，这样在每个存储周期内CPU和DMA都可以访问存储器
